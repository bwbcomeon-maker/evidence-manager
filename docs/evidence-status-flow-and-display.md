# 证据文件状态流转与页面展示说明

## 一、四种状态定义

| 状态码 | 中文 | 含义 | 是否计入门禁 |
|--------|------|------|--------------|
| **DRAFT** | 草稿 | 上传后仅保存、未正式提交 | 否（仅计入「上传进度」展示） |
| **SUBMITTED** | 已提交 | 已进入管理/审核流程 | 是 |
| **ARCHIVED** | 已归档 | 最终有效状态 | 是 |
| **INVALID** | 已作废 | 最终无效状态 | 否 |

- **门禁口径**（阶段完成度、申请归档）：只统计 **SUBMITTED + ARCHIVED**。
- **展示口径**（已上传数量）：统计 **DRAFT + SUBMITTED + ARCHIVED**；**INVALID** 不参与任何统计。

---

## 二、状态流转规则（后端强制校验）

```
                    [上传]
                       ↓
  ┌─────────┐    提交     ┌──────────┐    归档     ┌──────────┐
  │  DRAFT  │ ──────────► │ SUBMITTED │ ──────────► │ ARCHIVED │
  └────┬────┘             └─────┬─────┘             └──────────┘
       │                        │ 作废                    (终态)
       │ 物理删除                ↓
       │                  ┌──────────┐
       └─────────────────►│ INVALID  │
                          └──────────┘
                               (终态)
```

- **DRAFT**：只能 → **提交** 变为 SUBMITTED；草稿可 **物理删除**（从库中删除记录）。
- **SUBMITTED**：只能 → **归档** 或 **作废**；不能回退为草稿。
- **ARCHIVED / INVALID**：终态，不可再变更。

---

## 三、删除 vs 作废（业务语义）

| 操作 | 适用状态 | 含义 | 接口 |
|------|----------|------|------|
| **删除** | 仅 **DRAFT** | 物理删除证据记录，不可恢复 | `DELETE /api/evidence/{id}` |
| **作废** | 仅 **SUBMITTED** | 将状态改为 INVALID，需填写作废原因，记录作废人/时间 | `POST /api/evidence/{id}/invalidate` |

- **归档**：单条证据没有「取消归档」；项目维度有「申请归档」，会把项目下所有有效证据视为已归档使用。
- 已归档（ARCHIVED）证据：不可删除、不可作废，仅可查看/下载。

---

## 四、页面显示位置与操作入口

### 4.1 证据详情页（`/evidence/detail/:id`）

| 内容 | 位置 | 说明 |
|------|------|------|
| **当前状态** | 卡片头部标题右侧 | 胶囊 Badge：草稿 / 已提交 / 已归档 / 已作废 |
| **提交** | 底部操作区 | 仅当 **草稿** 且非「从证据管理入口进入」时显示；点击后变为已提交 |
| **删除** | 底部操作区 | 仅当 **草稿** 且非只读入口时显示；物理删除 |
| **作废** | 底部操作区 | 仅当 **已提交** 且当前用户有作废权限时显示；需填写作废原因 |
| **下载** | 底部操作区 | 所有状态均显示 |
| **预览** | 底部操作区 | 当媒体区未内嵌展示（如 Office/其他类型）时显示；图片/视频/PDF 已在顶部媒体区展示则不显示 |

- **从「按项目查看证据」进入**：视为只读，不显示提交/删除/作废，仅预览+下载。

### 4.2 项目详情页 · 证据 Tab（`/projects/:id`，证据管理 Tab）

| 内容 | 位置 | 说明 |
|------|------|------|
| **阶段内证据列表** | 每个阶段展开后的模板项下 | 网格展示缩略图/文件卡片 |
| **单条状态标签** | 每条证据缩略图/卡片上（如右下角或 overlay） | 小标签：草稿 / 已提交 / 已归档 / **已作废**（红色等） |
| **删除/作废入口** | 每条证据卡片上的垃圾桶图标 | 同一按钮：**草稿** → 点击为「删除」确认；**已提交** → 点击为「作废」确认；**已归档/已作废** 不显示该按钮 |
| **已 x/需 y 份** | 每个模板项右侧 | 仅统计 **门禁口径**（SUBMITTED+ARCHIVED），草稿与已作废不计入「已」 |

- 从「按项目查看证据」进入项目时（`from=evidence-by-project`）：证据网格**不展示**删除/作废按钮，避免误操作。

### 4.3 上传结果弹层（项目详情页 · 上传证据后）

| 内容 | 位置 | 说明 |
|------|------|------|
| **当前状态** | 结果卡片内 | 标签：草稿 / 已提交 等 |
| **提交** | 结果区按钮 | 仅当结果为 **草稿** 时显示；点击后该条变为已提交并刷新列表 |
| **删除** | 结果区按钮 | 仅当结果为 **草稿** 时显示；物理删除并关闭弹层 |
| **作废** | 结果区按钮 | 仅当结果为 **已提交** 时显示；弹窗填写作废原因后作废 |

---

## 五、与阶段完成度、归档的关系

- **阶段完成度 / 申请归档**：只认 **SUBMITTED + ARCHIVED** 条数达到模板项要求（如「已 1/需 1 份」）。
- **草稿**：计入「已上传」展示（如已 1/需 1 份旁的展示口径），但**不**满足阶段完成与归档门禁。
- **已作废**：既不参与「已上传」展示，也不参与门禁；列表中仍可展示为「已作废」标签，便于审计与查看。

---

## 六、小结表

| 状态 | 可执行操作 | 详情页按钮 | 项目证据网格 |
|------|------------|------------|--------------|
| 草稿 | 提交、删除 | 提交、删除、下载（+ 按类型预览） | 状态标签「草稿」+ 垃圾桶（删除） |
| 已提交 | 作废 | 作废、下载（+ 按类型预览） | 状态标签「已提交」+ 垃圾桶（作废） |
| 已归档 | — | 仅下载（+ 按类型预览） | 状态标签「已归档」，无垃圾桶 |
| 已作废 | — | 仅下载（+ 按类型预览） | 状态标签「已作废」，无垃圾桶 |

上述逻辑与后端 `EvidenceStatus` 枚举及接口权限一致，前端通过 `getEffectiveEvidenceStatus`、`permissions.canSubmit` / `canInvalidate` 控制按钮显隐与接口调用。
